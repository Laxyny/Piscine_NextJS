# Choisir un framework selon la taille du projet

## Objectif de l’exercice
Comprendre quels frameworks JavaScript sont adaptés à :
- un petit projet
- un projet de taille moyenne
- un grand projet

---

## 1. Définitions (à lire avant l’exercice)

### Petit projet
Un petit projet se caractérise par :
- Durée courte
- Peu de fonctionnalités
- Peu ou pas de logique métier complexe
- 1 à 2 développeurs
- Mise en place rapide

**Exemples :**
- Site vitrine
- Landing page
- Blog simple
- Prototype / POC

---

### Projet de taille moyenne
Un projet moyen se caractérise par :
- Plusieurs pages ou fonctionnalités
- Logique métier modérée
- Connexion à une API ou une base de données
- 2 à 5 développeurs
- Besoin de structure, sans lourdeur excessive

**Exemples :**
- Dashboard
- Application SaaS simple
- E-commerce basique

---

### Grand projet
Un grand projet se caractérise par :
- Logique métier complexe
- Forte évolutivité
- Sécurité avancée
- Tests, CI/CD
- Équipe importante
- Maintenance long terme

**Exemples :**
- Application d’entreprise
- Plateforme à grande échelle
- Microservices
- Application avec plusieurs clients (web, mobile)

---

## 2. Énoncé de l’exercice

Vous disposez ci-dessous d’une liste de frameworks JavaScript.  
Pour chacun d’eux, indiquez s’il est adapté à :
- un petit projet
- un projet de taille moyenne
- un grand projet

**Consignes :**
- Mettez un **X** dans la ou les cases correspondantes
- Un framework peut être adapté à plusieurs tailles de projet
